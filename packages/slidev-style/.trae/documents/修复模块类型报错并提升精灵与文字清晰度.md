## 问题原因
- 类型报错来源：`Demo034ChinaMap2.vue:24` 引入的模块 `./mesh/map/chinaMap2` 内部又依赖 `@/type`（见 `components/mesh/map/chinaMap2.ts:4`）。若当前项目未为 `@` 配置别名（或该别名未被 TypeScript 程序纳入 `tsconfig.include`），TypeScript 无法解析该依赖，最终在上层文件显示“找不到模块或其相应的类型声明”。
- 模糊原因：
  - 渲染器未设置设备像素比，WebGL 画布在高 DPI 屏幕上按低分辨率绘制，整体（包含 Sprite/文字）显得发虚（`MeshTemplate.vue:229-231`）。
  - `three-spritetext` 与 `SpriteMaterial` 的贴图使用默认过滤与 mipmap，在小尺寸/缩放下会造成过度模糊（`chinaMap2.ts:79-87` 与 `Demo034ChinaMap2.vue:94-103`）。

## 修复方案
1) 解决类型/模块解析：
- 方案 A（推荐）：在项目的 `tsconfig.json` 中添加别名与包含范围
  - `compilerOptions.baseUrl` 指向工程根；`paths` 增加 `"@/*": ["<你的源码根>/*"]`。
  - `include` 加入 `projects/slidevs/**` 或具体到 `learn-threejs/**`，确保 `chinaMap2.ts` 被 TS 程序纳入。
- 方案 B（更快）：移除 `chinaMap2.ts` 中对 `@/type` 的别名引用，改为相对路径导入共享类型文件；若找不到共享类型文件，则就地内联最小类型（`Feature`、`PolygonCoords`、`Position`）满足本文件使用。

2) 提升清晰度：
- 在 `MeshTemplate.vue` 初始化渲染器时启用抗锯齿并设置像素比：
  - 把 `new THREE.WebGLRenderer()` 改为 `new THREE.WebGLRenderer({ antialias: true })`（`MeshTemplate.vue:229`）。
  - 在 `setSize` 之后增加 `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))`（建议上限 2 兼顾性能）。
- 优化圆点精灵贴图：在 `chinaMap2.ts` 载入纹理后设置过滤与 mipmap（`chinaMap2.ts:79-87` 之后）：
  - `texture.generateMipmaps = false`
  - `texture.minFilter = THREE.LinearFilter`
  - `texture.magFilter = THREE.LinearFilter`（如要更锐利可用 `NearestFilter`，但会偏像素风）
  - `texture.needsUpdate = true`
- 优化文字精灵：创建 `SpriteText` 后设置其材质贴图过滤（`Demo034ChinaMap2.vue:94-103`）：
  - `posName.material.map.generateMipmaps = false`
  - `posName.material.map.minFilter = THREE.LinearFilter`
  - `posName.material.map.magFilter = THREE.LinearFilter`
  - 适当提高 `SpriteText` 的逻辑字号再通过 `scale` 缩小，获取更高基础分辨率。

## 验证步骤
- 运行开发环境并打开对应幻灯页面，确认：
  - TypeScript 不再报 `chinaMap2` 模块类型错误。
  - 高 DPI 屏幕下整体、圆点与文字明显更清晰。
- 若仍模糊，进一步：
  - 检查 `pos.png` 分辨率是否过低，替换为更高分辨率资源并相应减小 `annotation.scale`。
  - 针对 `three-spritetext`，确认库版本与贴图过滤设置已生效。
