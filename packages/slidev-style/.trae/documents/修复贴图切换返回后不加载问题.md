## 问题原因分析
- 现状：`zhuan.ts` 在模块加载时就创建并导出同一个 `Mesh` 实例（`export default mesh`，见 c:\home\Projects\frontend\my-slidevs\projects\slidevs\learn-threejs\components\mesh\zhuan.ts:20）。`Demo016zhuan.vue` 直接引用该实例（见 c:\home\Projects\frontend\my-slidevs\projects\slidevs\learn-threejs\components\Demo016zhuan.vue:18）。
- 在 Slidev 中切页会触发组件卸载，`MeshTemplate.vue` 的 `onUnmounted` 会调用 `disposeThreeResources({ renderer, scene, controls })`（见 c:\home\Projects\frontend\my-slidevs\projects\slidevs\learn-threejs\components\MeshTemplate.vue:279），极有可能把场景里对象的几何/材质/纹理都释放。
- 返回该页时仍复用同一个已被释放的 `Mesh` 实例，导致纹理或材质处于失效状态，结果就是“贴图切走了，切回这一页不加载”。
- 另一个隐患：`aoMap` 需要 `uv2` 属性，`PlaneGeometry` 默认只有 `uv`。当前未设置 `uv2`（见 zhuan.ts:17），会让环境遮挡图不生效，虽不致命，但建议补上。

## 修改方案（最小改动、按页重建）
1. 将 `zhuan.ts` 改为“工厂函数”导出，每次进入页面都创建全新的几何/材质/纹理实例，避免复用已被释放的对象。
2. 在 `Demo016zhuan.vue` 中调用工厂函数获取实例并传给 `MeshTemplate`。
3. 补充 `uv2`，让 `aoMap` 正常生效。

## 具体改动
### 1) `components/mesh/zhuan.ts`
- 替换默认导出为工厂函数：
```ts
import * as THREE from 'three'
import zhuanImage from '@/assets/zhuan.awebp'

export function createZhuan() {
  const loader = new THREE.TextureLoader()
  const texture = loader.load(zhuanImage)
  texture.wrapS = THREE.RepeatWrapping
  texture.wrapT = THREE.RepeatWrapping
  texture.repeat.set(3, 3)
  texture.colorSpace = THREE.SRGBColorSpace

  const geometry = new THREE.PlaneGeometry(1000, 1000)
  const uv = geometry.attributes.uv
  geometry.setAttribute('uv2', new THREE.BufferAttribute(uv.array, 2))

  const material = new THREE.MeshBasicMaterial({
    map: texture,
    aoMap: texture,
  })

  return new THREE.Mesh(geometry, material)
}
```

### 2) `components/Demo016zhuan.vue`
- 引入并调用工厂函数：
```ts
import MeshTemplate from './MeshTemplate.vue'
import { createZhuan } from './mesh/zhuan'
import * as THREE from 'three'

const zhuan = createZhuan()
const pointLight = new THREE.PointLight(0xffffff, 10000)
pointLight.position.set(80, 80, 80)
```
- 模板部分保持不变（`object3d` 仍然传 `zhuan`）。

## 备选方案（如仍需共用单例）
- 在 `MeshTemplate.vue` 内对外部传入的 `object3d` 执行深拷贝并仅释放拷贝，避免影响原始实例。但材质的 `map`/`aoMap` 通常仍指向同一纹理，释放时需要手动跳过或重建纹理，复杂度更高，不推荐。

## 可能的边界情况
- 若 `@` 别名未指向正确的 `src`，或 `awebp` 在某环境解码失败，建议临时改成 `png/jpg` 资源做 A/B 验证。
- 如需提升画质，可在创建渲染器后设置 `texture.anisotropy = renderer.capabilities.getMaxAnisotropy()`；本问题与是否显示无关。
